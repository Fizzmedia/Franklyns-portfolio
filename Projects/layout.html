Reflection

I used Flexbox for the navigation bar because it is a one-dimensional layout. The navbar mainly arranges items along a single horizontal axis: the logo, navigation links, and sign-up button. Flexbox makes it easy to control alignment using justify-content and align-items. It also handles spacing efficiently with gap and allows the layout to switch direction when the screen size changes.

I used Grid for the page skeleton because it is a two-dimensional layout. The page contains rows and columns that need to work together. Grid's grid-template-areas feature makes the placement model very clear by naming areas such as header, sidebar, main, and footer. Grid also supports spanning, which allowed one gallery card to stretch across two columns.

A real layout that would benefit from switching tools is a photo gallery built with Flexbox. While Flexbox can wrap items, Grid provides better control over rows and columns and makes it easier to create consistent card sizes. For layouts that require precise placement and spanning, Grid is generally the better choice.
